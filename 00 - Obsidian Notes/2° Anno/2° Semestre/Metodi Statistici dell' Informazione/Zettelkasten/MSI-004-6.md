# Trasformazioni molti-a-uno: aggregazione e riduzione di alfabeto

Quando g non è una biiezione, **più valori** dell'alfabeto originale 𝒳 possono mapparsi nello **stesso valore** di 𝒴. Questo è il caso molti-a-uno: la cardinalità dell'alfabeto si riduce, |𝒴| < |𝒳|.

In questo scenario, le probabilità si **collassano** (sommano): p_Y(y) = ∑_{x: g(x)=y} p_X(x). Se X = -1 ha probabilità 0.3 e X = +1 ha probabilità 0.4, e la funzione è Y = |X| (valore assoluto), allora p_Y(1) = 0.3 + 0.4 = 0.7 (perché sia -1 che +1 mappano a 1).

Esempio completo: X ∈ {-2, -1, 0, 1, 2} con p_X = {0.1, 0.2, 0.3, 0.2, 0.1}. Sia Y = X² (al quadrato). Allora Y ∈ {0, 1, 4}:
- p_Y(0) = p_X(0) = 0.3
- p_Y(1) = p_X(-1) + p_X(1) = 0.2 + 0.2 = 0.4
- p_Y(4) = p_X(-2) + p_X(2) = 0.1 + 0.1 = 0.2

Il collasso di probabilità è **perdita di informazione**: data Y, non possiamo ricostruire X (sappiamo Y=1 ma non se X=-1 o X=+1). Questo motiva lo studio di statistiche sufficienti in inferenza: quale trasformazione mantiene abbastanza informazione?
