### Virtualizzazione e Architetture del Kernel

#### Meccanismi di virtualizzazione

Il meccanismo di virtualizzazione serve per supportare più politiche. In termini implementativi, gli strati più vicini alla macchina sono in assembly, mentre i livelli successivi utilizzano linguaggi a più alto livello per garantire la portabilità del sistema.

**Esemplificazione con l'emulazione software:**
- **Emulazione:** Si crea una sorta di macchina virtuale che funziona come la macchina reale, simulando l'hardware dal software.
  - Sistema operativo host
  - Applicazione emula l'hardware della macchina da virtualizzare
  - Installazione del sistema operativo su questa piattaforma

**Meccanismi di virtualizzazione:**
- **Virtualizzazione completa:** Il sistema guest non sa di essere virtualizzato, ma sfrutta al massimo il sistema nativo per offrire una piattaforma.
- **Paravirtualizzazione:** Il sistema guest collabora con la virtual machine manager tramite API speciali.
- **Hardware-assisted virtualization:** Le CPU forniscono istruzioni e meccanismi di supporto per facilitare la virtualizzazione.

**Tipologie di virtualizzazione:**
- **Tipo 1 (Type 1):** Virtual Machine Manager che poggia direttamente sull'hardware.
- **Tipo 2 (Type 2):** Sistema installato sopra un sistema host, come VMware o VirtualBox.

#### Architetture del Kernel

Il kernel può essere strutturato in diversi paradigmi di progettazione:
1. **Monolitico:** Il kernel è tutto un unico blocco che gestisce tutte le funzionalità.
2. **Stratificato:** Prevede una serie di strati con separazione delle responsabilità.
3. **Microkernel:** Minimizza la dimensione del kernel, spostando operazioni verso lo user space.
4. **Modulare:** Il kernel è modulare e può caricare componenti dinamicamente.

**Esempio di Linux:**
- **Monolitico:** Inizialmente monolitico, ma ora modulare con struttura non esattamente monolitica.
- **Microkernel addomesticato:** Alcune funzionalità vengono spostate nello user space per migliorare la sicurezza.

**Caso di Windows NT:**
- Prima versione microkernel, successivamente passata a un sistema più monolitico con Windows XP.
  
### Ciclo di vita dei processi

Il ciclo di vita di un processo è sequenziale e può essere diviso in stati:
1. **Created:** Il processo viene creato ma non ancora eseguito.
2. **Ready:** Il processo è pronto per essere eseguito, ma atteso dalla CPU.
3. **Running:** Il processo sta attualmente eseguendo istruzioni.
4. **Waiting:** Il processo attende risorse esterne o eventi specifici.
5. **Terminated:** Il processo ha terminato l'esecuzione.

**Process Control Block (PCB):**
- Struttura dati che mantiene le informazioni del processo, come ID, stato esecutivo, e utilizzo della memoria.
  
### Creazione dei processi

Il sistema operativo mette a disposizione funzionalità per la creazione di processi attraverso chiamate al kernel:
- **fork:** Crea un processo figlio completamente separato dal padre.
- **exec:** Il figlio esegue un altro programma, sostituendo l'area text e lo stack.

**Esempio di fork in codice:**
```c
#include <stdio.h>
#include <unistd.h>

int main() {
    pid_t pid = fork();

    if (pid == 0) { // Figlio
        printf("Sono il figlio, PID: %d\n", getpid());
    } else if (pid > 0) { // Padre
        wait(NULL); // Attende la fine del processo figlio
        printf("Sono il padre, PID: %d\n", getpid());
    } else {
        perror("fork");
        return 1;
    }

    return 0;
}
```
